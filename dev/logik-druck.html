Track Logik

Zustands-Flags

VariableBedeutung
_trackTimeoutnull = nicht geplant, sonst: setTimeout-Handle
_trackLaeufttrue = GPS-Anfrage läuft gerade (verhindert Doppelstart)

Idempotenz-Schutz (wichtig!)

⚠ trackStarten() muss idempotent bleiben

hafenSperrungAktualisieren() wird bei jedem Event-Save aufgerufen. Ohne Idempotenzschutz würde jede Wende / jedes Reffen den laufenden Track-Timer zurücksetzen → Punkte würden nie aufgezeichnet.

Guard: if (_trackTimeout !== null || _trackLaeuft) return;

Ablauf Tracking

FAHRT beginnt → trackStarten() Prüfung: _trackTimeout !== null ODER _trackLaeuft → return (idempotent) → trackPunktAufzeichnenUndPlanen() _trackLaeuft = true, _trackTimeout = null getCurrentPosition() Erfolg: _trackLaeuft = false SOG → Intervall berechnen intervall > 0: Punkt in track.points speichern _trackTimeout = setTimeout(trackPunktAufzeichnenUndPlanen, intervall) Fehler: _trackLaeuft = false _trackTimeout = setTimeout(trackPunktAufzeichnenUndPlanen, 60000) STOPP / Törn wechsel → trackStoppen() clearTimeout(_trackTimeout), _trackTimeout = null, _trackLaeuft = false
