2 · Backup-System (3 Ebenen)

1
Auto-Backup
segel_logbuch_autobackup
  • Wird bei jedem Speicher-Vorgang aktualisiert
  • Manuell wiederherstellbar über Banner
  • Enthält: timestamp, toerns, crew
2
Permanentes Backup
segel_logbuch_backup_permanent
  • Gleichzeitig mit Auto-Backup geschrieben
  • Überlebt SW-Cache-Löschungen (Updates)
  • Automatische Wiederherstellung beim Start
3
Manuelles Backup
segellogbuch_backup_*.json
  • Datei-Download via „Backup JSON"
  • Enthält: toerns + crew als JSON-Datei
  • Wiederherstellung über Datei-Import

4 · Laden / Speichern Ablauf

Törn speichern (toernSpeichern)

1
Alle Törns laden: ladeToerns() → JSON.parse aus KEY_TOERNS
2
Törn suchen: findIndex(t.tripId) → Update oder Append
3
Array zurückschreiben: speichereToerns() → JSON.stringify → KEY_TOERNS
4
Crew-Namen extrahieren und in globale KEY_CREW-Liste mergen (Duplikate werden entfernt)
5
autoBackupSpeichern() → schreibt gleichzeitig KEY_AUTOBACKUP und KEY_PERMANENT_BACKUP

Törn löschen (toernLoeschen)

1
Alle Törns laden, gelöschten herausfiltern: filter(t.tripId !== tripId)
2
Gefiltertes Array zurückschreiben
3
Crew-Namen bereinigen: nur Namen behalten die noch in verbleibenden Törns vorkommen

5 · Wiederherstellung beim App-Start

App-Start (app.js, Zeile ~1973) │ ├─ 1. Migration ausführen (einmalig, IIFE in storage.js) │ Alten Key "segel_logbuch_trips" → "segel_logbuch_toerns" │ Alten Backup-Key migrieren │ ├─ 2. permanentBackupPruefen() │ ├─ Törns vorhanden? → JA: Permanent-Backup aktualisieren → weiter │ └─ Törns vorhanden? → NEIN: Permanent-Backup laden │ ├─ Backup hat Daten? → Wiederherstellen → _datenWiederhergestellt = true │ └─ Kein Backup? → Leer starten │ ├─ 3. toernSelectAktualisieren() → Dropdown + Übersicht aufbauen │ ├─ 4. ladeAktivenToernId() │ ├─ ID gefunden + Törn existiert? → toernLaden(id) → letzten Stand wiederherstellen │ └─ Nicht gefunden? → tabInhaltToggeln() + hafenSperrungAktualisieren() │ ├─ 5. backupBannerPruefen() │ → Zeigt Banner wenn Auto-Backup existiert aber Hauptdaten fehlen │ └─ 6. pwaMigrationPruefen() → Zeigt Import-Modal wenn keine Daten und kein Backup vorhanden
